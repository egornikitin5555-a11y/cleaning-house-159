ПРОБЛЕМА:
---------
Пользователь сообщил, что при прокрутке сайта с мобильного устройства (Android)
всё ещё присутствует избыточное пустое пространство, которое приходится 
прокручивать, что ухудшает пользовательский опыт.

РЕШЕНИЕ:
--------
Применены АГРЕССИВНЫЕ оптимизации spacing для всех мобильных брейкпоинтов
с драматическим уменьшением padding, margin и gap для всех секций сайта.

Типичный сайт содержит:
  • 1 Hero секция
  • ~10 основных секций (advantages, services, pricing, etc.)
  • ~10 пар заголовков (section-title + section-subtitle)

┌─────────────────────────────────────────────────────────────────────────┐
│  УСТРОЙСТВА 480px (ТИПИЧНЫЙ СМАРТФОН)                                  │
└─────────────────────────────────────────────────────────────────────────┘

Hero оптимизация:
  • Padding низ:               24px
  • Hero gap:                   8px
  • Hero внутренние отступы:   40px
  ИТОГО Hero:                  72px

Секции (×10):
  • Padding каждой:    16px × 10 = 160px
  
Заголовки (×10):
  • Каждая пара:       32px × 10 = 320px

ОБЩАЯ ЭКОНОМИЯ @ 480px:  72 + 160 + 320 = 552px ≈ 3.5 экрана


┌─────────────────────────────────────────────────────────────────────────┐
│  УСТРОЙСТВА 360px (КОМПАКТНЫЕ СМАРТФОНЫ)                               │
└─────────────────────────────────────────────────────────────────────────┘

Hero оптимизация:
  • Padding низ:               16px
  • Hero gap:                  16px (доп. оптимизация)
  • Hero внутренние отступы:   40px
  ИТОГО Hero:                  72px

Секции (×10):
  • Padding каждой:    32px × 10 = 320px
  
Заголовки (×10):
  • Экономия минимальна (уже оптимизировано)

ОБЩАЯ ЭКОНОМИЯ @ 360px:  72 + 320 = 392px ≈ 2.4 экрана

   ДАЛЬНЕЙШИЕ РЕКОМЕНДАЦИИ
═══════════════════════════════════════════════════════════════════════════

Если пустое пространство всё ещё присутствует:

1. ИДЕНТИФИЦИРОВАТЬ КОНКРЕТНУЮ СЕКЦИЮ
   → Открыть DevTools на мобильном
   → Найти элемент с избыточным padding/margin
   
2. ПРОВЕРИТЬ СПЕЦИФИЧНОСТЬ CSS
   → Убедиться, что media queries применяются
   → Проверить порядок каскада CSS
   
3. ДОПОЛНИТЕЛЬНАЯ ОПТИМИЗАЦИЯ
   → Можно уменьшить отступы ещё сильнее
   → Минимум: 4px (half of --space-xs)
   
4. АНАЛИЗ КОНТЕНТА
   → Проверить margin у параграфов (p { margin-bottom: 16px })
   → Проверить gap в grid/flex контейнерах
